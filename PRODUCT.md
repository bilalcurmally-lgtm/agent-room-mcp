# Product

## Register

product

## Users

Developers and technical leads who run multiple AI coding agents (Codex, Claude, Cursor) against shared projects. They use the dashboard at their desk, often for long sessions, to monitor agent conversation, assign work, and record decisions without editing JSON files.

## Product Purpose

Agent Room is a local control room for multi-agent coordination. The dashboard makes the shared room legible: read the feed, see agent status, manage tasks and decisions, and tell the room what to do next. Success means the user can orient in seconds and act without hunting through forms.

## Brand Personality

Calm, precise, trustworthy. A monitoring station, not a marketing page. Information-dense when needed, quiet by default.

## Anti-references

Late-90s web UI: grid backgrounds, uppercase tracked eyebrows on every section, nested accordions with forms inside forms, browser `prompt()` dialogs, six filter fields in the header, cream-tinted AI-default palettes, ghost cards with border plus heavy shadow.

## Design Principles

1. Feed first: the room conversation is the primary canvas; everything else is one click away.
2. Progressive disclosure: show one line to start; expand only when the user asks for protocol fields or task actions.
3. Earned density: compact like Linear or Raycast, but readable; no decorative chrome.
4. Dark default for evening monitoring; low glare, high contrast text.
5. Familiar affordances: sidebar nav, filter drawer, inline actions; do not reinvent standard patterns.

## Accessibility & Inclusion

WCAG 2.2 AA contrast on body text and controls. Respect `prefers-reduced-motion`. Keyboard-accessible nav, drawer, and forms. Semantic HTML and ARIA labels on icon-only controls.
